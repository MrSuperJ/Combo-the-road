ES5 构造对象的套路


    function CustomObject1 (x,y) {
      this.x = x
      this.y = y
    }  
    CustomObject.prototype.method = function () {
      return `${this.x} + ${this.y}`
    }
对照： 

    function CustomObject2 (x,y) {
      this.x = x
      this.y = y
      this.method = function(){
        return `${this.x} + ${this.y}` 
      }
    }
区别： 

    var o = new CustomObject1
    var p = new CustomObject2

    o.__proto__.method 存在
    p.__proto__ 为{}，不存在 method方法
结论： 使用new操作符， 生成的对象，__proto__ 指向 构造函数的prototype属性
